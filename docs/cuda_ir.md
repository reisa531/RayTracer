## CUDA IR 说明文档

#### 概述
本文档描述了把 CPU 构建的场景转化为 GPU 可读的数据链路的中间表示。中间表示需要去除所有 `Trait` 和智能指针，把各个 `Material` 和 `Texture` 转化为直接可读的具体类型并列成线性表，同时拍平 `BVH` 树。IR 暂时由 `.ir` 文件存储，由 Rust 前端创建，CUDA 后端读取。

#### Texture
Texture 段开始前，先出现 Header 行：
```
TEXTURE
SIZE = ...
```
其中，`...` 处是材质数目 `n`。
随后 `n` 行，每行 11 个数，第一个为类型编号（1 - 单色，2 - 棋盘，3 - 图片，4 - Perlin），接下来 9 个为三个 Texture 颜色（单色 - 仅第一个颜色有效；棋盘 - 仅前两个颜色有效；图片与 Perlin - 均无效；即使颜色无效，也必须用 0 来占位），最后为资源编号（目前仅对图片有效，为图片的资源编号，其他类型均为 0 占位符）。

#### Material
Material 段开始前，先出现 Header 行：
```
MATERIAL
SIZE = ...
```
其中，`...` 处是质地数目 `n`。
随后 `n` 行，每行 7 个数，第一个为类型编号（1 - 漫反射，2 - 金属，3 - 透射，其他的后续再添加），第二个为材质编号（目前仅对漫反射有效，其他的均为 0 占位符），随后三个是颜色参数（目前仅对金属有效，表示金属颜色，其他的均为 0 占位符），随后为两个参数（对于金属，第一个表示 `fuzz`；对于透射，第一个表示 `refractive_index`；第二个字段为预留的）。

#### Hittable
Hittable 段开始前，先出现 Header 行：
```
HITTABLE
SIZE = ...
```
其中，`...` 处是对象数目 `n`。
随后 `n` 行，每行 22 个数，第一个为类型编号（1 - 球体，2 - 三角形，3 - 四边形），随后 18 个 为 6 个向量的各个参数，第一个向量代表图形原点（球体 - 球心，三角形、四边形 - 一个顶点），后两个代表边向量（球体 - 占位符；三角形和四边形 - 两个邻边的向量），第四个代表移动向量，最后两个是预留向量（对于四边形，第五个向量是 `w` 字段；其他的暂时为预留）；随后三个参数，第一个为球体半径（仅对球有效），后两个表示材质（Texture）编号和质地（Material）编号（编号即为在对应段中出现的次序，从 0 开始编号）。

#### BVH
BVH 段开始前，先出现 Header 行：
```
BVH
SIZE = ...
```
其中，`...` 处是对象数目 `n`。
随后 `n` 行，每行 9 个数，前两个为当前节点的左右孩子编号（若无，使用 -1 占位），随后六个表示当前节点的包围盒，依次是 x 最小值、x 最大值、y 最小值、y 最大值、z 最小值和 z 最大值。最后为节点对应的对象编号（仅对叶子节点有效，否则为 -1 占位）。

#### Resource
Resource 段开始前，先出现 Header 行：
```
RESOURCE
SIZE = ...
```
其中，`...` 处是对象数目 `n`。
随后 `n` 行，每行开头为一整数，表示资源类型（1 - 图片），随后若为图片，出现一个字符串，表示图片路径。
